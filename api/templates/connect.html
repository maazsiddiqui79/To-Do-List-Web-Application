About Projects

To-Do List Application


Go ToDo is a fully responsive and minimalistic To-Do List application developed to help users organize tasks efficiently. It allows users to create, update, delete, and prioritize tasks with a clean interface. The app is structured with a clear separation between the front-end and back-end, ensuring scalability and maintainability.

Key Features:
- Add, update, and delete tasks
- Visual task prioritization with styling cues
- Responsive and mobile-friendly design
- Backend logic handled using Flask routes and forms
- Clean separation of concerns with dynamic templating using Jinja2
- Task storage and manipulation using SQLAlchemy ORM

Frontend:
HTML5 and CSS3 are used to structure content layout and provide custom styling for form inputs, cards, and task lists. Bootstrap 5 ensures responsive design and provides prebuilt UI components. Jinja2 templating is used to dynamically render HTML content using server-side variables and control logic.

Backend:
The backend is powered by Flask, which handles HTTP routes and form submissions, renders responses, and integrates easily with Jinja templates and SQLAlchemy. SQLAlchemy serves as the Object-Relational Mapper (ORM) to securely and efficiently interact with the database. It defines the Task model and allows easy CRUD operations using Python code instead of raw SQL.

Database:
SQLite is used for development and local testing, while the system is fully compatible with PostgreSQL for production environments. SQLAlchemy handles all database interactions and makes migrations and scaling straightforward.

Deployment & Hosting:
The front-end is hosted on Vercel for fast, reliable, and scalable deployment. GitHub integration allows for continuous deployment workflows. Vercel is ideal for static hosting or routing requests to a Flask backend hosted elsewhere (like Render or Railway).

Development Tools:
Visual Studio Code is used for code development. Git and GitHub are employed for version control and collaborative development. A Python virtual environment ensures all project dependencies are isolated and properly managed.

Technologies Used:
Languages: HTML5, CSS3, Python
Frameworks: Flask (Backend), Bootstrap (Frontend)
Templating: Jinja2
ORM & Database: SQLAlchemy with SQLite (development) / PostgreSQL (production-ready)
Version Control & CI/CD: Git, GitHub
Hosting: Vercel (frontend), with external support for Flask backend
